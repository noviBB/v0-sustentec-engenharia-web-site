# Docs

Code-agnostic knowledge and processes for this project. **Code-specific** guidance lives in the repo-root [CLAUDE.md](../CLAUDE.md) (commands, file paths, quirks).

## Start here

If you're new to this codebase, read in this order:

1. [conventions.md](conventions.md) — how we name things, where files go, the `"use client"` policy.
2. [i18n.md](i18n.md) — the single most important contract: every user-facing string goes through `t("key")`.
3. [design-system.md](design-system.md) — shadcn + Tailwind 4 + design tokens.
4. [workflows.md](workflows.md) — worktrees, branch flow, v0 auto-deploy.
5. [seo-and-accessibility.md](seo-and-accessibility.md) — metadata, lang, alt-text.
6. [roadmap.md](roadmap.md) — known gaps and what's deferred.
