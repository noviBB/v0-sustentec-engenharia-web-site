## Commands

Package manager: **pnpm** (pnpm-lock.yaml is the source of truth).

- `pnpm dev` — Next.js dev server on http://localhost:3000
- `pnpm build` — production build (note: `typescript.ignoreBuildErrors: true` in next.config.mjs)
- `pnpm start` — serve the production build
- `pnpm lint` — **currently broken**: script is `eslint .` but `eslint` is not in package.json and no config exists. See [docs/roadmap.md](docs/roadmap.md).
- `pnpm dlx shadcn@latest add <component>` — add a shadcn primitive into `components/ui/` (style: `new-york`, baseColor: `neutral`, icons: `lucide`).

There is no test runner. There are no CI workflows.

## Architecture

Single-page bilingual marketing site. [app/layout.tsx](app/layout.tsx) wraps a `LanguageProvider`, Geist fonts (Sans + Mono), and Vercel Analytics (production only) around the page tree. [app/page.tsx](app/page.tsx) composes section components from [components/](components/) in order: `Header`, `HeroSection`, `StatsSection`, `ServicesSection`, `TrackerSection`, `ValuesSection`, `ContactSection`, `Footer`. Navigation is in-page anchors (`#home`, `#services`, `#tracker`, `#about`, `#contact`) — there are no other routes.

The only cross-cutting concern is **i18n**: every user-facing string is keyed and looked up via `t("key")` from [lib/language-context.tsx](lib/language-context.tsx). PT is the default language and the fallback; EN is opt-in via the language switcher in the `Header`.

## Key patterns

- **Import alias**: `@/...` resolves to repo root (see [tsconfig.json](tsconfig.json) paths).
- **Class merging**: use `cn()` from [lib/utils.ts](lib/utils.ts) (clsx + tailwind-merge). Never concatenate Tailwind classes by hand.
- **Exports**: named function exports (`export function Foo()`). Avoid default exports for components (Next.js page/layout files excepted — those must be default).
- **Client vs server**: every component currently starts with `"use client"`. Keep that default unless the component is pure presentational AND removing the directive doesn't break a child's hook usage.
- **Icons**: lucide-react.
- **Forms**: react-hook-form + zod (already in deps). The current contact form does NOT use them — it's plain `useState` and `console.log`s the payload. See [docs/roadmap.md](docs/roadmap.md).
- **Theme**: `next-themes` is installed and a `theme-provider.tsx` exists, but the site is light-only right now.

## Component conventions

- [components/ui/](components/ui/) — shadcn/Radix primitives. Generated by `shadcn add`. Do not add business logic here.
- [components/](components/) (flat, no subfolders) — business / section components (e.g. [hero-section.tsx](components/hero-section.tsx), [contact-section.tsx](components/contact-section.tsx)). File names are kebab-case; the exported component is PascalCase.
- [hooks/](hooks/) — custom hooks, `use-*` prefix. Current: `use-mobile`, `use-toast`.
- [lib/](lib/) — shared utilities and context (`utils.ts`, `language-context.tsx`).

## Adding a translation key

Edit [lib/language-context.tsx](lib/language-context.tsx). The `translations` object has two top-level keys: `pt` and `en`. **A new key MUST be added to both dictionaries in the same change** — there is no fallback chain across languages, and the value returned for a missing key is the key itself. PT is authored first (source of truth); EN is translated from it.

Never hardcode user-facing strings in JSX. If you find a hardcoded string, move it to the dictionary as part of your change.

## Known quirks (preserve unless explicitly told to remove)

- [next.config.mjs](next.config.mjs) sets `typescript.ignoreBuildErrors: true` — TS errors don't block builds. Type checking should still pass; the flag is a v0 default safety net.
- [next.config.mjs](next.config.mjs) sets `images.unoptimized: true` — Next.js image optimization is disabled. Tracked in [docs/roadmap.md](docs/roadmap.md).
- The contact form has no backend. Submitting just `console.log`s. Tracked in [docs/roadmap.md](docs/roadmap.md).
- v0 sandbox files are gitignored (`__v0_*`, `.snowflake/`, `.v0-trash/`). Don't commit anything matching those patterns.
- This repo is linked to v0 project `prj_hp4OvygiRerka65XShcKsqg686cu`. **Merging to `main` triggers a v0 / Vercel auto-deploy.** See [docs/workflows.md](docs/workflows.md).

## Branch workflow (worktrees)

Branches are checked out under `./.worktrees/<branch-name>` (gitignored):

```
git worktree add ./.worktrees/<branch> -b <branch> main   # create
git worktree list                                         # list
git worktree remove ./.worktrees/<branch>                 # cleanup
```

Don't `cd` into a worktree and commit in the main checkout simultaneously — work in one or the other.
